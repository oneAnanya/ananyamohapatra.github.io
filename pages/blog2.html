Normalization in Databases

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. Hereâ€™s how the student table can be normalized through various normal forms (NF):

Unnormalized Form (UNF)

Initially, the data may be unorganized:

Name Student ID Course Duration
Ankita 101 PGDCA 6 months
Ananya 102 Full Stack 6 months
Archita 103 BI 8 months

First Normal Form (1NF)

1NF ensures each column has atomic (indivisible) values, and each record is unique.

Name Student ID Course Duration
Ankita 101 PGDCA 6 months
Ananya 102 Full Stack 6 months
Archita 103 BI 8 months

Second Normal Form (2NF)

2NF removes partial dependencies; all non-key columns must depend on the entire primary key. Here, since "Student ID" is the primary key and the table is already in 1NF, no further changes are needed because there are no partial dependencies.

Third Normal Form (3NF)

3NF removes transitive dependencies; non-key columns must not depend on other non-key columns. We split the table into two tables: "Students" and "Courses".

Students Table:

Student ID Name
101 Ankita
102 Ananya
103 Archita

Courses Table:

Student ID Course Duration
101 PGDCA 6 months
102 Full Stack 6 months
103 BI 8 months

Boyce-Codd Normal Form (BCNF)

BCNF is a stricter version of 3NF; it ensures no non-trivial functional dependencies of attributes on something other than a superkey. Our tables already comply with BCNF as there are no such dependencies.

Fourth Normal Form (4NF)

4NF deals with multi-valued dependencies. A table is in 4NF if it is in BCNF and has no multi-valued dependencies.

Example of Multi-valued Dependency:

Let's say a student can enroll in multiple courses and can have multiple hobbies.

Unnormalized Table:

Student ID Name Course Hobby
101 Ankita PGDCA Painting
101 Ankita Full Stack Painting
101 Ankita PGDCA Reading
101 Ankita Full Stack Reading
102 Ananya Full Stack Dancing
102 Ananya BI Dancing
102 Ananya Full Stack Singing
102 Ananya BI Singing

Here, both courses and hobbies are independent of each other but depend on the Student ID.

To convert this into 4NF:

1. Student-Course Table:

Student ID Name Course
101 Ankita PGDCA
101 Ankita Full Stack
102 Ananya Full Stack
102 Ananya BI

2. Student-Hobby Table:

Student ID Name Hobby
101 Ankita Painting
101 Ankita Reading
102 Ananya Dancing
102 Ananya Singing

Fifth Normal Form (5NF)

5NF deals with join dependencies. A table is in 5NF if it is in 4NF and cannot be decomposed into any smaller tables without losing data.

Example of Join Dependency:

Suppose we have the following unnormalized table that records which teacher teaches which course to which student:

Unnormalized Table:

Student ID Teacher Course
101 Mr. Sharma PGDCA
102 Mrs. Das Full Stack
103 Mr. Singh BI
101 Mr. Sharma Full Stack
102 Mrs. Das BI

To convert this into 5NF, we decompose it into three tables:

1. Student-Course Table:

Student ID Course
101 PGDCA
101 Full Stack
102 Full Stack
103 BI

2. Course-Teacher Table:

Course Teacher
PGDCA Mr. Sharma
Full Stack Mrs. Das
BI Mr. Singh

3. Student-Teacher Table:

Student ID Teacher
101 Mr. Sharma
101 Mrs. Das
102 Mrs. Das
103 Mr. Singh

Now there are no redundant dependencies, and all data is preserved accurately without redundancy. The table is in Fifth Normal Form (5NF).
